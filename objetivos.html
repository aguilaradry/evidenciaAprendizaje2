Objetivos del Proyecto

1. Implementar un sistema de control de versiones eficiente: Utilizar Git y GitHub para gestionar el código fuente del proyecto, asegurando la trazabilidad, colaboración efectiva y control de cambios en cada etapa del desarrollo.

2. Establecer un pipeline de CI/CD con GitHub Actions: Configurar un flujo automatizado que integre pruebas automatizadas, análisis estático del código y despliegue continuo, garantizando la calidad del software y una entrega rápida y confiable.

3. Automatizar pruebas del software: Diseñar e implementar pruebas automatizadas para validar la funcionalidad del código y minimizar errores durante las etapas de desarrollo y despliegue.

4. Realizar un despliegue continuo en un entorno en la nube: Implementar el software en un servicio de alojamiento en la nube, asegurando su accesibilidad y rendimiento óptimo en un entorno real.

5. Documentar el proyecto de manera integral: Crear y mantener documentación detallada del flujo de trabajo DevOps, incluyendo el diseño del pipeline, configuración de GitHub Actions, y la justificación de las decisiones técnicas tomadas durante el desarrollo.

6. Presentar el repositorio de GitHub como entregable final: Estructurar el repositorio con buenas prácticas, incluyendo un archivo README claro y descriptivo, registros de cambios, y scripts necesarios para facilitar la replicación y comprensión del flujo de trabajo DevOps implementado
